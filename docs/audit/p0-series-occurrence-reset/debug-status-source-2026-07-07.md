## Root Cause

1. Serien-Expansion erzeugt für index > 0 virtuelle IDs (`masterId::YYYY-MM-DD`).
2. Der Master-`assist_visits`-Datensatz trägt nach Durchführung des Ankertermins `canonical_status=completed`.
3. **Overlay/Enrichment** (`overlayVisitDispositionListFromAssignments`, `resolveLiveAssignment`, Portal-Assignment-Overlay) hat Master-Status auf alle Occurrences gelegt.
4. Fix: `resetVirtualOccurrenceListItem` / `neutralizeFutureOccurrenceListItem` neutralisieren Workflow, Proof, Doku, Tasks für virtuelle und fälschlich abgeschlossene Zukunftstermine.
